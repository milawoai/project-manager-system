# ESLint 全局规则说明

本文档记录了项目中重要的 ESLint 规则配置。

## 🚫 **禁止在函数内使用 require()**

### 规则描述
- **规则**: `@typescript-eslint/no-require-imports`
- **级别**: `error`
- **说明**: 禁止在函数内部使用 `require()` 语句

### 正确的导入方式

#### ✅ 正确的做法
```typescript
// 在文件顶部进行导入声明
import axios from 'axios'
import fs from 'fs'
import path from 'path'

function myFunction() {
  // 直接使用导入的模块
  const result = axios.get('http://example.com')
  return result
}
```

#### ❌ 错误的做法
```typescript
function myFunction() {
  // 禁止在函数内使用 require()
  const axios = require('axios')
  const result = axios.get('http://example.com')
  return result
}
```

### 配置文件位置
- **文件**: `.eslintrc.cjs`
- **规则配置**:
```javascript
rules: {
  '@typescript-eslint/no-require-imports': 'error',
  'no-restricted-imports': [
    'error',
    {
      name: 'require',
      message: '禁止在函数内使用 require()，请在文件顶部进行导入声明'
    }
  ]
}
```

### 原因和好处
1. **代码可读性**: 所有的依赖在文件顶部明确列出
2. **维护性**: 更容易识别和管理依赖关系
3. **Tree Shaking**: 现代打包工具能更好地进行优化
4. **一致性**: 保持统一的代码风格
5. **安全性**: 避免动态导入可能导致的安全问题

### 常见场景处理

#### 动态导入场景
如果确实需要动态导入（例如条件加载），使用 `import()`：
```typescript
// 动态导入模块
async function loadModule(condition: boolean) {
  if (condition) {
    const { default: heavyModule } = await import('./heavy-module')
    return heavyModule
  }
  return null
}
```

#### Node.js 特定模块
如果需要使用 Node.js 特定模块，仍然需要在顶部导入：
```typescript
import { exec } from 'child_process'

async function runCommand() {
  return new Promise((resolve, reject) => {
    exec('ls -la', (error, stdout, stderr) => {
      if (error) reject(error)
      else resolve(stdout)
    })
  })
}
```

## 其他重要规则

### Vue 相关规则
- `vue/require-default-prop`: `off` - 允许 props 无默认值
- `vue/multi-word-component-names`: `off` - 允许单字组件名

### TypeScript 相关规则
- `@typescript-eslint/no-explicit-any`: `off` - 允许使用 `any` 类型
- `@typescript-eslint/explicit-function-return-type`: `off` - 允许不显式声明函数返回类型

## 规则变更历史

### 2024-01-20
- 新增 `@typescript-eslint/no-require-imports` 规则
- 新增 `no-restricted-imports` 规则，专门限制 `require()` 使用
- 要求所有导入声明必须在文件顶部完成
